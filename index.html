XSLT er et deklarativt sprog

Another useful explanation I found in the Pro XAML with C#: On Declarative In declarative programming, the source code is written in a way that expresses the desired outcome of the code with little or no emphasis on the actual implementation. On Imperative Imperative programming is the opposite of declarative programming. If declarative programming can be thought of as declaring what the desired outcome is, imperative programming can be viewed as writing lines of code that represent the instructions of how to achieve the desired outcome.
